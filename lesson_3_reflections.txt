Q: When would you want to use a remote repository rather than keeping all your work local?
A: When you are working on it from multiple locations, with multiple people and/or as a backup

Q: Why might you want to always pull changes manually rather than having Git automatically stay up-to-date with 
   your remote repository?
A: It can override the changes you are making to a specific branch you are working on

Q: Describe the differences between forks, clones, and branches. When would you use one instead of another?
A: fork = copy of someone else which you can edit on your own version with a github-link between them
   clone = as fork, but without the connection
   branche = copy of the master locally on which you can work to add a new feature
   
Q: What is the benefit of having a copy of the last known state of the remote stored locally?
A: You can solve merge conflicts or work offline while having the latest copy

Q: How would you collaborate without using Git or GitHub? What would be easier, and what would be harder?
A: Sharing files throuhg dropbox or drive or working on the same server, but making changes makes it a lot harder 

Q: When would you want to make changes in a separate branch rather than directly in master? What benefits 
   does each approach have?
A: When adding a new feature, fixing large bugs, experimenting.
